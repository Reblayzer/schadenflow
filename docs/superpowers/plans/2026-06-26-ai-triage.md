# AI Triage (sub-project 4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an advisory AI-triage step — a reviewer triggers triage on a free-text claim and gets a summary + suggested category + missing-info flags; nothing is auto-applied, a reviewer confirms.

**Architecture:** A `triage/` package with a `TriageService` interface, a deterministic `MockTriageService` (default, used in all tests), and a config-gated real `AnthropicTriageService` (Anthropic Java SDK). `ClaimService` gains a `triage(...)` read path and an `updateCategory(...)` confirm path; `ClaimController` exposes `POST /api/claims/{id}/triage` (transient) and `PATCH /api/claims/{id}` (apply category + summary). `Claim.category` becomes an enum; a `triage_summary` column is added.

**Tech Stack:** Java 21, Spring Boot 3.4.1, Spring Security (from SP3), Anthropic Java SDK 2.34.0, Flyway, PostgreSQL, JUnit 5, Mockito, Testcontainers.

## Global Constraints

- **Package root:** `ch.sumex.schadenflow`. Java 21 / Spring Boot 3.4.1 / Maven.
- **Existing types (use exactly):** `Role` enum in `ch.sumex.schadenflow.claim` (`ANSPRUCHSTELLER`, `SACHBEARBEITER`, `ADMIN`); `ClaimState` enum in `ch.sumex.schadenflow.claim` (`EINGEREICHT`, `IN_PRUEFUNG`, `GENEHMIGT`, `ABGELEHNT`, `AUSBEZAHLT`); typed errors nested in `ch.sumex.schadenflow.shared.DomainException` (`NotFoundError`, `ValidationError`, `ForbiddenError`, `IllegalTransitionError`); envelope `ch.sumex.schadenflow.shared.ApiResponse` with `ApiResponse.ok(data)` / `ApiResponse.error(code, message)`; authenticated principal `ch.sumex.schadenflow.auth.AuthenticatedUser(UUID userId, String username, Role role)` read via `@AuthenticationPrincipal`.
- **Category model:** fixed `Category` enum (`EnumType.STRING`): `ARZTKOSTEN`, `MEDIKAMENTE`, `SPITAL`, `ZAHNARZT`, `THERAPIE`, `HILFSMITTEL`, `SONSTIGES`.
- **Triage output:** `{ summary, suggestedCategory, missingInfoFlags }`, advisory only, never auto-applied. `MissingInfoFlag` enum: `MISSING_AMOUNT`, `VAGUE_DESCRIPTION`, `MISSING_DATE`, `MISSING_PROVIDER`.
- **Authorization:** triage + confirm are reviewer-only (`SACHBEARBEITER`/`ADMIN`); claimant → 403 `FORBIDDEN`. Triage allowed only while the claim is `EINGEREICHT` or `IN_PRUEFUNG` (else 422 `VALIDATION`). `PATCH` is not state-gated.

- [ ] **Step 4: Create `triage/MockTriageService.java`** (deterministic; no Spring annotation — it is wired by `TriageConfig`)

```java
package ch.sumex.schadenflow.triage;

import java.math.BigDecimal;
import java.util.ArrayList;
import java.util.List;
import java.util.Locale;
import java.util.regex.Pattern;

import ch.sumex.schadenflow.claim.Category;

public class MockTriageService implements TriageService {

    private static final int VAGUE_DESCRIPTION_MIN_LENGTH = 20;
    private static final Pattern DATE_LIKE = Pattern.compile("\\d{4}-\\d{2}-\\d{2}|\\d{1,2}\\.\\d{1,2}\\.\\d{2,4}");

    @Override
    public TriageResult triage(TriageInput input) {
        String description = input.description() == null ? "" : input.description();
        String lower = description.toLowerCase(Locale.ROOT);
        Category category = suggestCategory(lower);
        List<MissingInfoFlag> flags = flags(input, description, lower);
        String summary = summarize(description);
        return new TriageResult(summary, category, flags);
    }

    private Category suggestCategory(String lower) {
        if (lower.contains("zahn")) {
            return Category.ZAHNARZT;
        }
        if (lower.contains("apotheke") || lower.contains("medikament")) {
            return Category.MEDIKAMENTE;
        }
        if (lower.contains("spital") || lower.contains("klinik")) {
            return Category.SPITAL;
        }
        if (lower.contains("therapie")) {
            return Category.THERAPIE;
        }
        if (lower.contains("brille") || lower.contains("hilfsmittel")) {
            return Category.HILFSMITTEL;
        }
        if (lower.contains("arzt")) {
            return Category.ARZTKOSTEN;
        }
        return Category.SONSTIGES;
    }

    private List<MissingInfoFlag> flags(TriageInput input, String description, String lower) {
        List<MissingInfoFlag> flags = new ArrayList<>();
        if (input.amount() == null || input.amount().compareTo(BigDecimal.ZERO) == 0) {
            flags.add(MissingInfoFlag.MISSING_AMOUNT);
        }
        if (description.length() < VAGUE_DESCRIPTION_MIN_LENGTH) {
            flags.add(MissingInfoFlag.VAGUE_DESCRIPTION);
        }
        if (!DATE_LIKE.matcher(description).find()) {
            flags.add(MissingInfoFlag.MISSING_DATE);
        }
        if (!(lower.contains("arzt") || lower.contains("apotheke") || lower.contains("klinik")
                || lower.contains("spital") || lower.contains("optiker") || lower.contains("therapeut"))) {
            flags.add(MissingInfoFlag.MISSING_PROVIDER);
        }
        return flags;
    }

    private String summarize(String description) {
        String trimmed = description.strip();
        if (trimmed.isEmpty()) {
            return "Keine Beschreibung vorhanden.";
        }
        int firstStop = trimmed.indexOf('.');
        String firstSentence = firstStop > 0 ? trimmed.substring(0, firstStop + 1) : trimmed;
        return firstSentence.length() > 200 ? firstSentence.substring(0, 200) : firstSentence;
    }
}
```

- [ ] **Step 4: Create `triage/AnthropicTriageService.java`.** It builds a prompt that constrains the category to the enum and asks for a strict JSON object, calls the Messages API, extracts the text, and parses it leniently. Parsing is split into a package-private `parseResponseJson(String)` so it is unit-testable without the SDK.

```java
package ch.sumex.schadenflow.triage;

import java.util.ArrayList;
import java.util.List;

import ch.sumex.schadenflow.claim.Category;
import com.anthropic.client.AnthropicClient;
import com.anthropic.models.messages.Message;
import com.anthropic.models.messages.MessageCreateParams;
import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;

public class AnthropicTriageService implements TriageService {

    private static final ObjectMapper MAPPER = new ObjectMapper();
    private static final String CATEGORY_VALUES =
            "ARZTKOSTEN, MEDIKAMENTE, SPITAL, ZAHNARZT, THERAPIE, HILFSMITTEL, SONSTIGES";
    private static final String FLAG_VALUES =
            "MISSING_AMOUNT, VAGUE_DESCRIPTION, MISSING_DATE, MISSING_PROVIDER";

    private final AnthropicClient client;
    private final String model;

    public AnthropicTriageService(AnthropicClient client, String model) {
        this.client = client;
        this.model = model;
    }

    @Override
    public TriageResult triage(TriageInput input) {
        String prompt = buildPrompt(input);
        try {
            MessageCreateParams params = MessageCreateParams.builder()
                    .model(model)
                    .maxTokens(1024L)
                    .addUserMessage(prompt)
                    .build();
            Message response = client.messages().create(params);
            String text = response.content().stream()
                    .flatMap(block -> block.text().stream())
                    .map(t -> t.text())
                    .reduce("", String::concat);
            return parseResponseJson(text);
        } catch (TriageUnavailableException ex) {
            throw ex;
        } catch (RuntimeException ex) {
            throw new TriageUnavailableException("Triage provider call failed", ex);
        }
    }

    private String buildPrompt(TriageInput input) {
        return """
                Du bist ein Triage-Assistent für Krankenversicherungs-Schadenfälle.
                Analysiere den folgenden Schadenfall und antworte AUSSCHLIESSLICH mit einem JSON-Objekt
                der Form {"summary": string, "suggestedCategory": string, "missingInfoFlags": string[]}.
                suggestedCategory MUSS einer dieser Werte sein: %s.
                missingInfoFlags darf nur diese Werte enthalten: %s.
                Titel: %s
                Beschreibung: %s
                Betrag: %s
                """.formatted(CATEGORY_VALUES, FLAG_VALUES,
                        input.title(), input.description(), String.valueOf(input.amount()));
    }

    TriageResult parseResponseJson(String json) {
        try {
            JsonNode root = MAPPER.readTree(json);
            String summary = root.path("summary").asText("");
            Category category = parseCategory(root.path("suggestedCategory").asText(""));
            List<MissingInfoFlag> flags = new ArrayList<>();
            for (JsonNode flag : root.path("missingInfoFlags")) {
                parseFlag(flag.asText("")).ifPresent(flags::add);
            }
            return new TriageResult(summary, category, flags);
        } catch (RuntimeException | com.fasterxml.jackson.core.JsonProcessingException ex) {
            throw new TriageUnavailableException("Could not parse triage response", ex);
        }
    }

    private Category parseCategory(String value) {
        try {
            return Category.valueOf(value);
        } catch (IllegalArgumentException ex) {
            return Category.SONSTIGES;
        }
    }

    private java.util.Optional<MissingInfoFlag> parseFlag(String value) {
        try {
            return java.util.Optional.of(MissingInfoFlag.valueOf(value));
        } catch (IllegalArgumentException ex) {
            return java.util.Optional.empty();
        }
    }
}
```

- `TriageService` has a deterministic mock (default, used in all tests) and a config-gated `AnthropicTriageService` that fails safe (503 `TRIAGE_UNAVAILABLE`, no secret leakage); provider selected by `SCHADENFLOW_TRIAGE_PROVIDER`.
- README documents the provider toggle; CI runs on the mock.
