# Infra & Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the Schadenflow monorepo skeleton — a Spring Boot backend with a health endpoint wired to PostgreSQL, an Angular + Material frontend, all orchestrated by Docker Compose, with a green GitHub Actions CI pipeline.

**Architecture:** Monorepo with `/backend` (Spring Boot, Maven, Java 21), `/frontend` (Angular standalone + Material), and `/infra` (Dockerfiles, compose). The backend exposes `GET /api/health` returning the project's standard envelope. PostgreSQL wiring is verified with a Testcontainers integration test. CI runs backend (`mvn verify`) and frontend (`npm ci && build && test`) jobs in parallel.

**Tech Stack:** Java 21, Spring Boot 3.4.x, Maven 3.9, PostgreSQL 16, Testcontainers, Angular 19 (standalone + signals), Angular Material, Node 22, Docker Compose, GitHub Actions.

## Global Constraints

- **Java version:** 21 (exact: backend compiles and runs on JDK 21).

**Files:**

**Interfaces:**
- Consumes: `SchadenflowApplication` from Task 1.
- Produces: a Spring Boot app that connects to PostgreSQL via env vars `SPRING_DATASOURCE_URL`, `SPRING_DATASOURCE_USERNAME`, `SPRING_DATASOURCE_PASSWORD`. Proves boot-against-Postgres with an integration test (`*IT`).

> **Why Failsafe (Step 1b):** the Spring Boot parent only configures the
> Surefire plugin, which runs `*Test` classes. Integration tests named `*IT`
> are run by the Failsafe plugin, which the parent does **not** bind by default.
> Without it, `mvn verify` would silently skip `ApplicationContextIT`. Adding
> Failsafe keeps slow Docker-backed tests out of the fast `mvn test` loop while
> still running them in `mvn verify`.

- [ ] **Step 1a: Add dependencies to `backend/pom.xml`** — insert these `<dependency>` blocks inside `<dependencies>`, after the `spring-boot-starter-web` block:

```xml
            <scope>runtime</scope>
        </dependency>
```

- [ ] **Step 1: Create `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up JDK 21
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: "21"
          cache: maven
      - name: Build and test
        working-directory: backend
        run: mvn -B verify

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Node 22
        uses: actions/setup-node@v4
        with:
          node-version: "22"
          cache: npm
          cache-dependency-path: frontend/package-lock.json
      - name: Install dependencies
        working-directory: frontend
        run: npm ci
      - name: Lint and build
        working-directory: frontend
        run: npm run build
      - name: Unit tests (headless)
        working-directory: frontend
        run: npm test -- --watch=false --browsers=ChromeHeadless
```
